OVERVIEW

What is KTOx?

KTOx is a full-spectrum network penetration testing and security education toolkit built around the ARP protocol — the layer-2 glue that maps IP addresses to MAC addresses on every LAN. It rivals tools like bettercap, Responder, and ettercap individually, while offering something none of them do: a complete paired defensive suite alongside every offensive module.

⚡ ARP FACT
ARP has no authentication. When your computer asks "who has 192.168.1.1?", any machine on the network can answer — and your computer will believe it. This design decision from 1982 is still the foundation of most LAN-level attacks today.
📡 PROTOCOL TIP
ARP operates entirely within a broadcast domain. This means VLAN segmentation is one of the most effective defenses — ARP packets literally cannot cross VLAN boundaries without routing through a firewall first.
🔬 WHY PYTHON
bettercap is written in Go and compiled — fast, but opaque. KTOx is pure Python using Scapy for packet crafting. Every line is readable, auditable, and extensible without a compiler. On constrained hardware like Raspberry Pi running Kali ARM, Python plays nicely with the architecture.

🧠 NETWORK FACT
An ARP table on a standard home router holds roughly 512–2048 entries. KTOx's ARP Flood can generate thousands of fake entries per second using randomized source MACs, overflowing the table and causing the switch to enter "fail-open" mode — broadcasting all frames to all ports instead of selectively forwarding. This is how a single machine can eavesdrop on an entire switched network.

⚡ LLMNR FACT
LLMNR (Link-Local Multicast Name Resolution) was introduced in Windows Vista as a fallback when DNS fails. When a Windows machine can't find a hostname via DNS, it broadcasts a multicast query to the entire LAN segment — no authentication, no verification. KTOx responds claiming to be that hostname. The victim's machine then automatically sends NTLMv2 credentials to authenticate. This happens with zero user interaction — just a mistyped hostname or a network resource that doesn't exist.

🥷 STEALTH FACT
Most IDS systems detect ARP attacks by looking for rate anomalies — a host sending far more ARP packets than normal. Ghost mode keeps KTOx below the noise floor of normal ARP traffic (typically 1–10 packets/minute on an idle LAN). The idle injection adds benign ARP WHO-HAS requests between attack packets, making the traffic pattern look like normal host discovery rather than an active attack. MAC rotation means even if an alert fires, the source address has already changed.
⚙
CUSTOM PROFILE
Select [U] Stealth Mode → [4] Custom to configure every parameter manually: exact ppm cap, min/max jitter bounds, rotation interval in seconds, and whether to enable idle injection. The stealth engine's stealth_send() context manager wraps any sendPacket() call transparently — attack modules don't need to know stealth is active.

🍓
VENV WITH SUDO (Raspberry Pi / Kali)
When using a Python venv, sudo strips the venv from PATH. Use the full path instead:

sudo ./venv/bin/python3 ktox.py --gui

This preserves all installed packages while running as root — required for raw socket access on Linux.
